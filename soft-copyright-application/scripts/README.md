# 前 1500 行 / 后 1500 行源码生成脚本说明

软著申请在源码超过 3000 行时，需提交《前1500行》与《后1500行》两个文件，且起止须满足：

- **前 1500 行**：从某模块的**开始代码**起头；
- **后 1500 行**：以说明书**最后一个功能**的**最末行代码**收尾。

## 参数说明

| 参数 | 含义 |
|------|------|
| **项目根路径** | 目标项目根目录（含源码的仓库根） |
| **FILE_ORDER** | 与说明书叙述顺序一致的源码文件相对路径列表；按此顺序拼接后统计「非空行」 |
| **BACK_END_AT** | 后 1500 行的结束位置：`(文件相对路径, 行号)`，表示该文件只取到第 N 行（含），该行应为「说明书最后功能」的最后一行代码 |
| **输出目录** | 生成 `代码_前1500行.txt`、`代码_后1500行.txt` 的目录 |

## 逻辑简述

1. 按 `FILE_ORDER` 依次读取每个文件，收集**非空行**，得到整份源码行序列；
2. **前 1500 行**：取该序列的前 1500 行写入 `代码_前1500行.txt`；
3. **后 1500 行**：构造「后半段顺序」：`FILE_ORDER` 中把 `BACK_END_AT` 对应文件放到最后，且该文件只读到指定行号；按此顺序再收集非空行，取**最后 1500 行**写入 `代码_后1500行.txt`。

## 如何确定 FILE_ORDER 与 BACK_END_AT

- **FILE_ORDER**：与说明书「功能模块」顺序一致，通常为主包/核心模块 → 配置/工具 → 示例/测试；
- **BACK_END_AT**：在说明书中找到**最后一个功能章节**（如「5.9 原始交易发送模块」），定位该功能对应的源码文件及该功能实现的**最后一行代码**，记为 `(文件路径, 行号)`。
